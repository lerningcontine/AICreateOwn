# 三个从 0 到 1 项目

## 项目 1：个人知识库问答助手

### 目标

上传自己的学习资料、文章、PDF，让助手能基于资料回答问题，并给出引用。

### 功能范围

MVP：

- 支持 Markdown / PDF / Word 文档。
- 自动切块并存入向量库。
- 用户提问后返回答案和引用。
- 不确定时拒答。

进阶：

- 支持多知识库。
- 支持文档版本。
- 支持引用跳转。
- 支持评测面板。

### 技术栈建议

- Python。
- FastAPI。
- Chroma 或 FAISS。
- 任意大模型 API。
- 任意 Embedding 模型。
- Streamlit 或简单 Web 前端。

### 验收

- 50 个问题正确率超过 80%。
- 回答必须带引用。
- 无资料问题拒答率超过 90%。

## 项目 2：AI 客服助手

### 目标

基于产品 FAQ、售后政策、物流规则，自动回答用户问题。

### 功能范围

MVP：

- FAQ 问答。
- 售后政策问答。
- 用户问题分类。
- 查不到转人工。

进阶：

- 接入订单查询工具。
- 多轮追问。
- 用户满意度反馈。
- 客服话术风格微调。

### 技术重点

- Prompt 语气控制。
- RAG 知识库。
- 工具调用。
- 安全和权限。
- 线上日志评测。

### 验收

- 高频问题自动解决率超过 60%。
- 错误承诺率低于 2%。
- 人工客服认为可用。

## 项目 3：行业报告生成助手

### 目标

输入行业资料、公司资料、竞品信息，自动生成结构化报告初稿。

### 功能范围

MVP：

- 上传资料。
- 自动提取要点。
- 生成报告大纲。
- 按章节生成内容。
- 输出 Markdown 报告。

进阶：

- 自动生成 PPT 大纲。
- 引用来源。
- 多版本改写。
- 风格模板。

### 技术重点

- 长文档处理。
- 多步骤 Prompt 编排。
- RAG 引用。
- 事实核查。
- 输出格式控制。

### 验收

- 报告结构完整。
- 核心事实能追溯来源。
- 人工编辑时间减少 30% 以上。

## 项目通用开发节奏

第 1 周：

- 明确用户和场景。
- 写需求文档。
- 准备 20 条真实样例。
- 跑通最小 Demo。

第 2 周：

- 做 RAG 或 Prompt 优化。
- 建立评测集。
- 记录错误。

第 3 周：

- 做界面和日志。
- 小范围试用。
- 迭代 2-3 版。

第 4 周：

- 做上线检查。
- 算成本。
- 写商业化假设。
- 找真实用户试用。
